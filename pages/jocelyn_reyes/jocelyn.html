What I Worked On

As I mentioned above, at DataDog I worked on the corporate website, which uses a static site generator as their framework. I was able to work on the website's case studies pages.
One part of this project was to edit the further readings section, in other words the related posts, at the bottom of the page and make sure it didn't only link to DataDog's blog post, which was how it originally was. They wanted that section to be linked to DataDog's resource pages as well as external pages. Due to the company's growth this change was needed because there is now more information that can be linked in the further readings section. In order to acheive this new front-matter had to be added to the markdown file and this front-matter was called on the html page. The second part of this project was to make sure each quote on a case study page has a gradient block on the side which fit the company's brand color. The final part was to make the sure the quotation marks weren't hard corded, this was a necessary change because due to more quotes being included in new case study pages it's crucial to cite the author. To make the second and third parts of the project possible there was new front-matter added, the scss file was edited in order to remove the hard-coded quotes, a default gradient was added, and the cite was hard-coded instead. The css was directly added to the html file and lastly I added the quotation marks as well as the author to the pages' markdown file.
